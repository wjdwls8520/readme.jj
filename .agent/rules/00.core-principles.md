# The 10 Golden Rules (Strictly Apply)

1. Separation of Concerns (SoC):
   - Strictly separate UI (View), Business Logic (Model/Service), and Data Fetching.
   - Never mix logic inside UI components. Use patterns like Container-Presenter or Custom Hooks.

2. Component Reusability (Atomic Design):
   - Break down UIs into the smallest reusable units (Atoms -> Molecules -> Organisms).
   - Avoid monolithic components that are hard to maintain.

3. Unidirectional Data Flow:
   - Ensure data flows from parent to child. Avoid Props Drilling.
   - Use Context API, Redux, or Zustand for global state if depth > 2.

4. DRY Principle (Don't Repeat Yourself):
   - Extract repeated logic into utility functions or hooks.
   - Do not copy-paste similar code blocks.

5. SOLID Principles:
   - SRP: One module should have one reason to change.
   - OCP: Open for extension, closed for modification.
   - DIP: Depend on abstractions, not concretions.

6. Immutability:
   - Never mutate state directly. Use spread operators (...) or libraries like Immer.
   - Ensure functional purity where possible.

7. Type Safety (TypeScript):
   - No 'any' type. Define explicit interfaces and types for all props and state.
   - Use Generics for reusable logic.

8. RESTful / GraphQL Standards:
   - Use proper HTTP methods and status codes.
   - Keep API resource naming consistent and predictable.

9. Standardized Folder Structure (FSD/MVC):
   - Organize files by feature or domain, not just file type.
   - Follow the framework's recommended best practices (e.g., Next.js App Router).

10. Naming Convention:
    - Variables: camelCase (descriptive, e.g., isUserLoggedIn)
    - Components: PascalCase
    - Constants: UPPER_SNAKE_CASE
    - Functions: Verb-noun pairs (e.g., fetchData, handleSubmit)
